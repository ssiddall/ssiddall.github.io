CMS feature set from Denison University

and other liberal arts colleges (CLAC)

 

This statement of our tools list is not intended to describe the basic CMS.  We know that this audience knows a great deal about current commercial CMS products so this is a list of basic features that we are not getting from current products.  Denison and most of the other schools in the Consortium of Liberal Arts Colleges (CLAC, http://liberalarts.org) are using Blackboard, WebCT, Prometheus or others. 

 

 

Content management

 

  • Ability to update an object without having to delete it first.

 

  • Fuller use of drag and drop interface for document management

 

  • More flexible page layouts that don’t go so far as to confuse

 

  • Better handling of HTML imports, including pages that contain images and metadata

 

  • Stronger set of faculty and administrator tools for managing course archives

 

  • Automated creation of basic student portfolios that track experiences in courses, offering students the opportunity to augment/expand this archival content to create a holistic view of their education; requires accessible storage of archived course content, rosters, etc.

 

  • Ability to deliver high-bandwidth media in ways that don’t slow the learning experience

 

  • Content delivery to PDAs and PPCs

 

  • Handle diacritical marks and other characteristics of Western languages, as well as non-Western language characters

 

  • Built-in equation editor for use in discussion boards

 

  • Built-in search tools that provide a high level of control, data mining (e.g., searching within a course, or among courses)

 

  • Access controls that help us comply easily with current interpretation of copyrights

 

 

Integration with campus systems

 

We view the OKI as a key resource to integrate into our campus’ portal, not as a replacement for the campus portal itself.  We must be able to integrate it with our administrative systems (the major vendors, PeopleSoft, SCT, Datatel, Jenzabar, etc. ) on the back end and with our portals on the front end.

 

  • Courses generated for all courses taught; both “opt-in” (default is course unavailable) and “opt-out” (default is course available) models available

 

  • Links to official course descriptions published in course catalog in addition to syllabi or other ad hoc descriptions prepared by instructors

 

  • Automated create/delete users, including specification of roles; ability to define groups for deletion (e.g., graduating seniors)

 

  • Real time updates for registrations and drop/add changes

 

  • Single signon features using variety of campus standards (LDAP, Kerberos, etc).

 

  • Prominence for local library resources, and the ability to link to and from library resources with ease (including authentication schemes), including electronic reserves

 

  • Integrate communication features (mail, discussions, calendar, announcements, etc) as XML channels that can be easily brought into a portal; “You have new postings to read”

 

  • Well defined APIs that enable us to plug-in other resources to the CMS interface

 

 

Communication and collaboration

 

  • Ability to access class e-mail lists from outside the CMS; make it easy to integrate communication tools with portals (noted above)

 

  • Ability to work collaboratively on a document, synchronously or asynchronously, with annotations, tracking changes, side-by-side comparisons, etc

 

  • Ability to add links to existing documents; e.g., students adding links to pages posted in CMS by instructor

 

  • Video and audio chat capabilities (including multicasting)

 

Assessment management

 

  • Ability to manage grade book within the CMS, or to export-edit-import in spreadsheet format.

 

  • Better methods for “real” authentication of student identification during testing

 

  • Self-assessment tools for review

 

  • Built-in basic statistical tools, both for assessment and student work with content

 

  • Options for assessing essays collaboratively or use of tools such as calibrated peer review (http://cpr.molsci.ucla.edu/cpr_info/index.asp)

 

  • Survey tools that allow for branching depending on responses

 

Compatibility

 

  • Adhere standards (such as IMS) for interoperability with other CMS
